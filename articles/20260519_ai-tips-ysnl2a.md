## 1. 議事録作成テンプレート

### プロンプト

```
以下の会議内容から、構造化された議事録を作成してください。

【会議情報】
- 会議名：[会議名を入力]
- 日時：[日時を入力]
- 参加者：[参加者を入力]

【会議内容（箇条書きメモでOK）】
[ここに会議中に取ったメモを貼り付け]

【出力形式】
1. 議題
2. 決定事項（具体的な内容と担当者、期限を明記）
3. 課題・懸念点
4. 次回のアクションアイテム（担当者と期限付き）
5. 次回会議の日程（決まっている場合）

読みやすく、関係者が一目で状況を把握できる形式でお願いします。
```

### 活用例

実際に私がプロジェクト会議で使った例です：

**入力した会議メモ：**
```
- 新機能の実装スケジュール確認
- 田中さん：API連携は来週完了予定
- 佐藤さん：UIデザインはまだ修正中、木曜までに完成させる
- テストは再来週から開始したい
- リリースは月末を目指す、ただしQAの状況次第
```

**ChatGPTの出力：**

実際にこのテンプレートを使うと、わずか30秒程度で読みやすい議事録が完成します。メモを整理する時間が劇的に短縮されました。

### カスタマイズのコツ

- 会社特有の議事録フォーマットがある場合は、「出力形式」部分をカスタマイズ
- 「担当者と期限を明記」を強調することで、曖昧さを排除
- 業界特有の用語がある場合は、プロンプトに「[業界名]の用語を使用」と追記

## 2. 技術仕様書作成テンプレート

### プロンプト

```
以下の機能について、開発チーム向けの技術仕様書を作成してください。

【機能概要】
[実装したい機能の概要]

【技術スタック】
[使用する技術・言語・フレームワーク]

【要件】
[具体的な要件をリスト形式で]

【出力内容】
1. 機能概要（非エンジニアでも理解できる説明）
2. 技術的アプローチ
3. データモデル設計（必要な場合）
4. API仕様（エンドポイント、メソッド、パラメータ）
5. エラーハンドリング
6. セキュリティ考慮事項
7. パフォーマンス要件
8. テスト方針

開発者が実装に迷わない、具体的で明確な仕様書にしてください。
```

### 活用例

この例では、ユーザー認証機能の仕様書を作成してみます：

**入力例：**
```
【機能概要】
ユーザーログイン機能の実装

【技術スタック】
- バックエンド：Node.js + Express
- データベース：PostgreSQL
- 認証：JWT

【要件】
- メールアドレスとパスワードでログイン
- トークンの有効期限は24時間
- パスワードは8文字以上で英数字混在必須
```

ChatGPTは、この情報から**API仕様、エラーケース、セキュリティ対策**まで含めた包括的な仕様書を生成してくれます。特に、見落としがちなセキュリティ要件やエッジケースを提案してくれる点が非常に有用です。

## 3. 提案書・企画書テンプレート

### プロンプト

```
以下の内容で、経営層や意思決定者向けの提案書を作成してください。

【提案の背景】
[なぜこの提案が必要か]

【提案内容】
[具体的に何をするのか]

【期待される効果】
[定量的・定性的な効果]

【必要なリソース】
[人員、予算、期間など]

【出力形式】
1. エグゼクティブサマリー（1-2段落で全体像）
2. 現状の課題分析
3. 提案の詳細
4. 期待される効果・ROI
5. 実施スケジュール
6. 必要リソースと予算
7. リスクと対応策
8. 結論と次のステップ

説得力があり、意思決定者が判断しやすい構成でお願いします。
数値データがある場合は表形式も活用してください。
```

### 実践ポイント

提案書では**説得力**が重要です。ChatGPTに以下を追加で指示すると、より効果的な提案書になります：

- 「定量的な効果を数値で示してください」
- 「競合他社の事例も参考に含めてください」
- 「想定される反論とその回答も用意してください」

私の経験では、ChatGPTが生成した提案書の骨子を使うことで、**資料作成時間が3時間から1時間に短縮**されました。残りの時間を数値の検証やプレゼン練習に充てられるようになったのは大きなメリットです。

## 4. ユーザーマニュアル作成テンプレート

### プロンプト

```
以下のシステム・ツールについて、エンドユーザー向けのマニュアルを作成してください。

【対象システム】
[システム名・機能名]

【想定ユーザー】
[ユーザーの属性、ITリテラシーレベル]

【説明したい機能】
[機能のリスト]

【出力要件】
1. 各機能の目的と概要
2. 操作手順（ステップバイステップ）
3. 画面キャプチャの挿入箇所を [画像: 説明] で明示
4. よくあるエラーとトラブルシューティング
5. Tips・注意事項
6. FAQ（よくある質問）

専門用語は最小限にし、初心者でも理解できる平易な文章でお願いします。
各ステップには番号を振り、操作の流れが明確になるようにしてください。
```

### マニュアル作成の工夫

マニュアルは**読者視点**が何より重要です。ChatGPTに以下のような追加指示を出すと、より親切なマニュアルになります：

```
- 各手順で「なぜこの操作が必要か」を簡潔に説明してください
- 失敗しやすいポイントには⚠️警告マークを付けてください
- 「次のステップに進む前に確認すべきこと」を各セクションの最後に追加してください
```

### 実際の成果

社内ツールのマニュアル作成で使用したところ、サポート問い合わせが**月間30件から10件に減少**しました。ユーザーが自己解決できる情報が充実したことが要因です。

## 5. 報告書・レポート作成テンプレート

### プロンプト

```
以下のデータ・情報から、上司や関係者向けの報告書を作成してください。

【報告期間】
[期間を指定]

【報告内容（データ・事実）】
[数値データ、実施内容、発生した事象など]

【報告の目的】
[何を伝えたいか、どんな判断を求めるか]

【出力形式】
1. サマリー（結論を先に）
2. 期間中の主な活動・成果
3. 定量データ（表やグラフで示す箇所を明示）
4. 課題と要因分析
5. 今後の対応方針
6. 次期の目標・計画

結論ファーストで、忙しい読者が最初の段落だけで全体像を把握できる構成にしてください。
データは可能な限り視覚的に整理してください。
```

### ビジネスでの活用実例

月次報告書の作成に使った例：

**入力データ：**
```
【報告期間】2024年1月

【報告内容】
- 新規顧客獲得：15社（目標12社）
- 売上：850万円（目標800万円）
- 解約：2社
- 問い合わせ対応：120件（前月比+20件）
- 新機能リリース：3件
```

ChatGPTは、このデータから**成果の評価、課題の洗い出し、改善提案**まで含めた報告書を生成します。特に、数値の背景にある要因分析の提案が有用です。

## プロンプトを効果的に使うための3つのコツ

### 1. 具体的な情報を与える

曖昧な指示では曖昧な結果しか得られません。以下を明確にしましょう：

- **読者は誰か**（経営層、開発者、一般ユーザーなど）
- **目的は何か**（承認を得る、情報共有、行動を促すなど）
- **出力形式**（箇条書き、段落、表形式など）

### 2. 段階的に改善する

一度の出力で完璧を目指さず、**対話しながら改善**していきます：

```
「セキュリティの部分をもっと詳しく説明してください」
「非エンジニア向けにもう少し平易な表現にしてください」
「具体例を3つ追加してください」
```

### 3. テンプレートをカスタマイズして保存

自社の文化や業界に合わせて、プロンプトを**カスタマイズ**しましょう：

- 社内の用語や略語を追加
- よく使う出力形式を標準化
- 業界特有の要件を組み込む

私は、カスタマイズしたプロンプトを**Notionにテンプレート集として保存**しています。必要なときにコピペするだけで、一貫した品質のドキュメントが作成できます。

## ドキュメント品質を向上させるチェックリスト

ChatGPTが生成した文書は、必ず以下をチェックしてから使用しましょう：

### ✅ 事実確認

- [ ] 数値データは正確か
- [ ] 固有名詞のスペルは正しいか
- [ ] 日付や期限は合っているか

### ✅ 文脈の整合性

- [ ] 文章の流れは自然か
- [ ] 論理的な矛盾はないか
- [ ] 読者の知識レベルに合っているか

### ✅ 目的との整合性

- [ ] 伝えたいメッセージは明確か
- [ ] 読者が取るべきアクションは明示されているか
- [ ] 不要な情報は削除されているか

### ✅ 見た目の調整

- [ ] 見出しの階層は適切か
- [ ] 箇条書きや表は効果的に使われているか
- [ ] 強調（太字など）は適切か
