# はじめに

ChatGPTを使っているけれど、「期待した答えが返ってこない」「毎回質問を繰り返している」という経験はありませんか?

実は、ChatGPTの性能を最大限引き出すには**プロンプト（指示文）の設計**が鍵となります。同じ質問でも、プロンプトの書き方次第で回答の質が劇的に変わるのです。

本記事では、私が業務で実際に使っている**プロンプトエンジニアリングのテクニック**を、具体例とともに解説します。この記事を読めば、ChatGPTから高品質な回答を引き出し、業務効率を大幅に改善できるでしょう。

## プロンプトエンジニアリングとは?

プロンプトエンジニアリングとは、AIに対する指示（プロンプト）を最適化し、望む出力を得るための技術です。

### なぜ重要なのか?

- **同じ質問でも、書き方で結果が変わる**: 曖昧な指示では曖昧な回答しか得られません
- **時間の節約**: 一度で的確な回答を得られれば、何度もやり直す必要がありません
- **AIの能力を最大化**: ChatGPTは強力ですが、適切な指示がなければ本来の力を発揮できません

## 基本原則：良いプロンプトの5つの要素

良いプロンプトには以下の5つの要素が含まれています。

### 1. 役割（Role）の指定

AIに「誰として振る舞うか」を指示します。

**悪い例：**
```
Pythonのコードをレビューしてください。
```

**良い例：**
```
あなたは10年以上の経験を持つシニアPythonエンジニアです。
以下のコードをレビューし、パフォーマンス、可読性、セキュリティの観点から改善点を指摘してください。
```

役割を明確にすることで、回答の専門性と視点が定まります。

### 2. 文脈（Context）の提供

背景情報を与えることで、的確な回答が得られます。

**悪い例：**
```
メールの文章を書いて。
```

**良い例：**
```
【状況】
- 取引先への納期遅延のお詫びメール
- 遅延理由：サプライヤーからの部品供給遅れ
- 新しい納期：当初より2週間遅れ
- トーン：丁寧かつ誠実に

上記の状況で、取引先に送る謝罪メールを作成してください。
```

### 3. 指示（Instruction）の具体化

何をしてほしいか、明確に伝えます。

**悪い例：**
```
データ分析について教えて。
```

**良い例：**
```
以下のステップで、初心者向けにデータ分析の流れを説明してください：
1. データ収集の方法
2. データクリーニングの重要性
3. 基本的な分析手法（平均、中央値、標準偏差）
4. 可視化の方法
5. 結果の解釈

各ステップは2-3行で、具体例を1つずつ含めてください。
```

### 4. 制約条件（Constraints）の設定

出力の形式、長さ、スタイルなどを指定します。

```
【制約条件】
- 文字数：300字以内
- トーン：カジュアル
- 形式：箇条書き
- 対象読者：非エンジニア
```

### 5. 出力形式（Output Format）の指定

どのような形式で出力してほしいかを明示します。

```
以下の形式で出力してください：

## タイトル
【概要】
（100字程度）

【詳細】
1. ポイント1
2. ポイント2
3. ポイント3

【まとめ】
（50字程度）
```

## 実践テクニック

### テクニック1：Few-Shot Learning（例示による学習）

具体例を示すことで、望む出力形式を伝えます。

```
以下の例に従って、商品レビューを分析してください。

【例】
レビュー：「配送が早くて助かりました。商品も期待通りでした。」
分析結果：
- 評価：肯定的（★4.5）
- 評価ポイント：配送速度、商品品質
- 改善点：特になし

【分析対象】
レビュー：「商品は良いけど、箱が潰れていた。梱包を改善してほしい。」
分析結果：
```

このテクニックは、データ整形やレポート作成に特に有効です。

### テクニック2：Chain of Thought（思考の連鎖）

ステップバイステップで考えさせることで、複雑な問題の精度を上げます。

```
以下の問題を、ステップごとに考えながら解いてください。

問題：ある店舗の月間売上が120万円、固定費が40万円、変動費率が30%です。
損益分岐点売上高を計算し、現在の営業利益も求めてください。

【回答手順】
1. まず、変動費を計算する
2. 次に、貢献利益を計算する
3. 損益分岐点売上高の公式を適用する
4. 最後に、現在の営業利益を計算する

各ステップで計算式と結果を明示してください。
```

### テクニック3：ペルソナ法

具体的なペルソナ（架空の人物像）を設定することで、ターゲットに合った回答を得ます。

```
以下のペルソナ向けに、クラウドサービスの導入メリットを説明してください。

【ペルソナ】
- 名前：田中太郎（50歳）
- 役職：中小企業の経営者
- ITリテラシー：メールとExcelは使える程度
- 課題：社内のデータ管理が紙ベースで非効率
- 懸念：セキュリティとコスト

専門用語は避け、具体的なメリットを3つ、各100字程度で説明してください。
```

### テクニック4：反復改善プロンプト

初回の出力を改善するためのフォローアップ指示を出します。

```
# 初回プロンプト
ブログ記事のタイトル案を5つ提案してください。
テーマ：リモートワークの生産性向上

# フォローアップ
上記の案の中から、最も読者の関心を引きそうなタイトルを1つ選び、
その理由を説明してください。さらに、そのタイトルを改善した
バージョンを3つ提案してください。
```

## 業務別プロンプト実例集

### コーディング支援

```
あなたは経験豊富なソフトウェアエンジニアです。

【タスク】
以下の要件を満たすPython関数を作成してください。

【要件】
- 機能：CSVファイルから特定の列を抽出し、欠損値を処理する
- 入力：CSVファイルパス、抽出する列名のリスト
- 出力：pandasデータフレーム
- 欠損値処理：数値列は平均値、文字列列は"Unknown"で補完

【制約】
- エラーハンドリングを含める
- docstringでドキュメント化
- 型ヒントを使用

【出力形式】
1. コード
2. 使用例
3. 想定されるエッジケースとその対処
```

### 文書作成

```
あなたはビジネスライティングの専門家です。

【タスク】
新しいプロジェクト提案書の要約（エグゼクティブサマリー）を作成してください。

【プロジェクト情報】
- プロジェクト名：社内業務システムのクラウド移行
- 目的：運用コスト30%削減、災害対策強化
- 期間：6ヶ月
- 予算：2,000万円
- 期待効果：年間運用費600万円削減、システム可用性99.9%達成

【要件】
- 長さ：300字程度
- 読者：経営陣（技術的詳細は不要）
- 含めるべき内容：目的、期待効果、投資対効果
- トーン：説得力があり、簡潔
```

### データ分析

```
あなたはデータアナリストです。

【データ】
以下は顧客アンケート結果の要約です：
- 回答者数：500名
- 満足度（5段階）平均：3.2
- リピート意向：40%
- 主な不満：「価格が高い」45%、「品揃えが少ない」30%、「配送が遅い」25%

【タスク】
1. この結果から読み取れる3つの重要なインサイトを抽出
2. 各インサイトに対する具体的な改善提案を1つずつ提示
3. 優先順位をつけ、その理由を説明

【出力形式】
各インサイトと提案は以下の形式で：
- インサイト：（50字以内）
- 改善提案：（100字以内）
- 優先度：高/中/低
- 理由：（80字以内）
```

## よくある失敗例と改善策

### 失敗例1：指示が曖昧

**❌ 悪い例：**
```
良いキャッチコピーを考えて。
```

**✅ 改善例：**
```
【商品情報】
- 商品：オーガニックコーヒー豆
- ターゲット：30-40代の健康志向の女性
- 特徴：フェアトレード、無農薬、深煎り
- 訴求ポイント：毎朝の贅沢な習慣

上記の商品のキャッチコピーを5つ提案してください。
各15字以内、商品の特徴と感情的なベネフィットを含めてください。
```

### 失敗例2：一度に複数のタスクを要求

**❌ 悪い例：**
```
マーケティング戦略を考えて、予算も計算して、実行スケジュールも作って。
```

**✅ 改善例：**
タスクを分割し、段階的に進めます。

```
# ステップ1
まず、新商品のマーケティング戦略の骨子を3つの柱で提案してください。
各柱につき100字程度で説明してください。

# ステップ2（最初の出力後）
上記の戦略案の第1の柱について、詳細な実行計画を立ててください。
施策内容、期間、必要リソースを含めてください。

# ステップ3（2回目の出力後）
その実行計画に基づいて、予算案を作成してください。
```

### 失敗例3：前提知識を共有していない

**❌ 悪い例：**
```
このコードを最適化して。
[コードだけ貼り付け]
```

**✅ 改善例：**
```
【背景】
- 用途：毎日100万件のログデータを処理
- 現在の問題：処理時間が30分かかり、業務に支障
- 環境：Python 3.9、メモリ8GB
- 制約：外部ライブラリの追加は可能

以下のコードを、処理時間を1/3以下にすることを目標に最適化してください。
最適化のポイントと、期待される効果も説明してください。

[コード]
```

## まとめ

プロンプトエンジニアリングは、AIを効果的に活用するための必須スキルです。本記事で紹介したテクニックをまとめます。

### 重要ポイント

1. **5つの要素を意識する**: 役割、文脈、指示、制約、出力形式
2. **具体的に指示する**: 曖昧さを排除し、期待する結果を明確に
3. **例を示す**: Few-Shot Learningで望む形式を伝える
4. **段階的に進める**: 複雑なタスクは分割して指示
5. **反復改善する**: 初回の出力をベースにブラッシュアップ

### 実践のステップ

1. まず、既存の業務で使っているプロンプトを見直す
2. 本記事のテンプレートを自分の業務に適用してみる
3. 結果を記録し、どの要素が効果的だったか分析する
4. 自分専用のプロンプトライブラリを構築する
