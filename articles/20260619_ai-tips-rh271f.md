## はじめに

ChatGPTをなんとなく使っているけれど、「思ったような回答が得られない」「時間をかけて質問しても期待外れ」という経験はありませんか?

実は、プロンプト(質問の仕方)を少し工夫するだけで、ChatGPTから得られる価値は劇的に変わります。本記事では、開発現場で実際に使える**即戦力プロンプト10選**を、具体例とともにご紹介します。

### この記事で得られる価値

- コピペで使える実践的なプロンプトテンプレート
- プロンプト設計の基本原則の理解
- 開発効率を2〜3倍に高める具体的手法
- よくある失敗パターンとその改善方法

それでは、実際に現場で使えるプロンプトを見ていきましょう!

## プロンプト設計の3つの基本原則

具体的なプロンプトに入る前に、効果的なプロンプトの基本原則を押さえておきましょう。

### 1. 役割を明確にする(Role)

ChatGPTに「あなたは〇〇です」と役割を与えることで、回答の質が向上します。

```
❌ 悪い例: Reactのコードを書いて
✅ 良い例: あなたは10年以上の経験を持つReact専門のシニアエンジニアです。
         TypeScriptとReact Hooksを使った保守性の高いコードを書いてください。
```

### 2. 文脈と制約を示す(Context & Constraints)

具体的な状況や制約条件を伝えることで、的確な回答が得られます。

### 3. 期待する出力形式を指定する(Format)

表形式、箇条書き、コードブロックなど、欲しい形式を明示しましょう。

## プロンプト効果を最大化する5つのコツ

### 1. 段階的に詳細化する

最初から完璧なプロンプトを作ろうとせず、ChatGPTとの対話を通じて徐々に精度を上げましょう。

```
1回目: 基本的な質問
2回目: 「もっと具体的に」「別の角度から」と追加質問
3回目: 最も欲しい情報に絞り込む
```

### 2. 出力例を示す(Few-shot Learning)

期待する出力の例を1〜2個示すと、ChatGPTが形式を理解しやすくなります。

```
以下の形式で出力してください:

【例】
関数名: calculateTotal
目的: 配列の数値を合計する
パラメータ: numbers (number[])
戻り値: number

このような形式で、以下の関数を説明してください...
```

### 3. 制約条件を明確にする

文字数、使用技術、対象読者など、制約を明示することで精度が上がります。

### 4. 複数案を求める

「3つの案を提示してください」と指定することで、より良い選択肢が得られます。

### 5. 評価基準を伝える

「パフォーマンス重視」「保守性重視」など、何を優先するか伝えましょう。

## よくある失敗パターンと改善策

### 失敗例1: 質問が曖昧すぎる

```
❌ 「Reactについて教えて」
✅ 「React 18のuseDeferredValueフックの使い所と、
   useTransitionとの使い分けを、具体例を交えて教えてください」
```

### 失敗例2: 文脈情報が不足

```
❌ 「このエラーを直して」
✅ 「Next.js 14のApp Routerで、サーバーコンポーネントから
   クライアントコンポーネントにpropsを渡す際に発生した
   以下のエラーを解決したいです: [エラー内容]」
```

### 失敗例3: 一度に多くを求めすぎる

```
❌ 「ECサイトを作るコード全部書いて」
✅ 「ECサイトの商品一覧ページのReactコンポーネントを、
   まずは静的データで表示する部分から作ってください」
```
