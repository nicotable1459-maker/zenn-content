## はじめに：カスタム指示で変わるAI活用の質

ChatGPTを日常的に使っているあなた、毎回同じような前置きや説明を繰り返していませんか?

「あなたはプロのエンジニアです。コードレビューをお願いします...」
「私は初心者なので、分かりやすく説明してください...」

こうした指示を毎回入力するのは時間の無駄です。**カスタム指示（Custom Instructions）**を活用すれば、ChatGPTがあなたの好みや背景を常に記憶し、より的確な回答を返してくれるようになります。

この記事では、実務で即使える具体的なカスタム指示の設定方法と、生産性を劇的に向上させるテクニックを紹介します。

### この記事で得られること

- カスタム指示の基本的な設定方法
- 職種別・用途別の実践的なカスタム指示例
- 生産性を最大化する応用テクニック
- よくある失敗パターンと改善方法

## カスタム指示とは？

カスタム指示は、ChatGPTに対してあなた自身の情報や回答スタイルの希望を事前に設定できる機能です。設定すると、すべての会話でその情報が自動的に考慮されます。

### 2つの設定項目

カスタム指示には以下の2つのセクションがあります：

1. **"What would you like ChatGPT to know about you?"**（あなたについて知っておいてほしいこと）
   - 職業、スキルレベル、興味関心など

2. **"How would you like ChatGPT to respond?"**（どのように回答してほしいか）
   - 回答スタイル、文章の長さ、専門用語の使用度など

## 実践！職種別カスタム指示テンプレート

### エンジニア向けカスタム指示

#### あなたについて知っておいてほしいこと

```
私はWebエンジニア（経験5年）です。
主にTypeScript、React、Node.jsを使った開発を行っています。
クリーンアーキテクチャとテスト駆動開発を重視しています。
最新のベストプラクティスとパフォーマンスに関心があります。
```

#### どのように回答してほしいか

```
- コード例は必ずTypeScriptで記述してください
- 型定義を明示的に含めてください
- セキュリティやパフォーマンスの考慮点があれば必ず指摘してください
- コードには簡潔なコメントを付けてください
- 複数の実装方法がある場合は、pros/consを比較してください
- 回答は技術的に正確であることを最優先してください
```

**効果例：**
設定前は「TypeScriptで書いてください」と毎回指定が必要でしたが、設定後は自動的にTypeScriptのコードが返ってくるようになります。

### マーケター向けカスタム指示

#### あなたについて知っておいてほしいこと

```
私はデジタルマーケターです。
BtoB SaaS企業でコンテンツマーケティングとSEOを担当しています。
データドリブンな意思決定を重視しています。
ターゲットは中小企業の経営者・マネージャー層です。
```

#### どのように回答してほしいか

```
- マーケティング戦略は具体的な数値目標とKPIを含めてください
- 施策のROI（投資対効果）を常に意識した提案をしてください
- BtoB特有の長い購買サイクルを考慮してください
- 専門用語は使っても構いませんが、必要に応じて補足説明を加えてください
- 実行可能なアクションプランの形で提案してください
```

### ライター・コンテンツクリエイター向けカスタム指示

#### あなたについて知っておいてほしいこと

```
私はフリーランスのテックライターです。
主にWeb開発やAI関連の技術記事を執筆しています。
読者は中級レベルのエンジニアとテクノロジーに関心がある非エンジニアです。
SEOとユーザビリティの両立を重視しています。
```

#### どのように回答してほしいか

```
- 文章は読みやすく、親しみやすいトーンで書いてください
- 専門用語を使う際は、初心者でも理解できるよう簡単に説明を加えてください
- 具体例や比喩を積極的に使って、抽象的な概念を分かりやすくしてください
- 記事構成の提案時は、見出しの階層構造を明確にしてください
- SEOキーワードの自然な配置を意識してください
```

## 応用テクニック：カスタム指示を最大限活用する

### 1. コンテキストの階層化

カスタム指示だけでなく、プロジェクト固有の情報は会話の冒頭で追加します。

```
【カスタム指示】：あなたの基本情報（常に適用）
↓
【会話冒頭】：プロジェクト固有の情報
↓
【個別の質問】：具体的なタスク
```

**実例：**

```
# 会話冒頭で追加する情報
今から新規ECサイトのReact実装について相談します。
- フレームワーク：Next.js 14（App Router）
- 決済：Stripe
- デザイン：Tailwind CSS
- 期限：2ヶ月

この前提で質問に答えてください。
```

### 2. 出力フォーマットの指定

回答のフォーマットを統一すると、情報の整理が格段に楽になります。

```
# カスタム指示に追加
技術的な質問への回答は以下の構造で提供してください：
1. 要約（2-3行）
2. 詳細な説明
3. コード例
4. 注意点・ベストプラクティス
5. 参考リンク（存在する場合）
```

### 3. 学習スタイルの明示

あなたの理解の仕方を伝えることで、より効果的な学習が可能になります。

```
# 学習スタイルの例
- 私は具体例から理解するタイプです。抽象的な説明の前に必ず実例を示してください
- 視覚的な図解やフローチャートの提案があれば教えてください
- 段階的に理解を深めたいので、基礎→応用の順で説明してください
```

## 生産性を3倍にする具体的な活用シーン

### シーン1：コードレビューの自動化

**カスタム指示に追加：**

```
コードレビューを依頼された際は、以下の観点で必ずチェックしてください：
1. バグや潜在的な問題
2. パフォーマンスの改善点
3. 可読性とメンテナンス性
4. セキュリティリスク
5. テストの網羅性
重要度順にリストアップし、それぞれに改善案を提示してください。
```

**使用例：**

```
# 単にコードを貼り付けるだけ
const fetchUsers = async () => {
  const res = await fetch('/api/users');
  return res.json();
}
```

ChatGPTは自動的に包括的なレビューを返してくれます。

### シーン2：ドキュメント作成の効率化

**カスタム指示に追加：**

```
技術ドキュメントを作成する際は、以下の構成を使用してください：
1. 概要（何ができるか）
2. 前提条件
3. ステップバイステップの手順
4. トラブルシューティング
5. よくある質問

コードブロックには言語名を明記し、シンタックスハイライトに対応してください。
```

### シーン3：ブレインストーミングの質向上

**カスタム指示に追加：**

```
アイデア出しを依頼された際は：
- 最低10個のアイデアを提供してください
- 実現可能性（低/中/高）を併記してください
- 各アイデアに簡単な実装方法を添えてください
- 型破りなアイデアも1-2個含めてください
```

## よくある失敗パターンと改善方法

### 失敗パターン1：情報過多

❌ **悪い例：**

```
私は東京在住の32歳男性で、大学は○○大学を卒業し、
趣味は読書と映画鑑賞で、好きな食べ物はラーメンで...
```

✅ **良い例：**

```
私はWebエンジニア（Node.js/React、経験5年）です。
クリーンコードとテスト駆動開発を重視しています。
```

**改善ポイント：**
- ChatGPTの回答に影響する情報だけを記載
- 個人的すぎる情報は不要

### 失敗パターン2：曖昧な指示

❌ **悪い例：**

```
分かりやすく説明してください
```

✅ **良い例：**

```
中級エンジニア向けに、具体的なコード例を含めて説明してください。
専門用語は使用可能ですが、複雑な概念には簡単な比喩を添えてください。
```

**改善ポイント：**
- 「分かりやすい」の基準を具体化
- ターゲットレベルを明示

### 失敗パターン3：矛盾する指示

❌ **悪い例：**

```
初心者向けに説明してください
高度な技術的詳細も含めてください
```

✅ **良い例：**

```
初心者が理解できる平易な言葉で説明してください。
技術的な詳細は「補足」セクションに分けて記載してください。
```

## カスタム指示のメンテナンス

カスタム指示は「設定したら終わり」ではありません。定期的な見直しが重要です。

### 月次レビューのチェックリスト

- [ ] 自分のスキルレベルや関心が変わっていないか
- [ ] 使用している技術スタックに変更はないか
- [ ] 回答の質に満足しているか
- [ ] 不要な制約や指示が含まれていないか
- [ ] 新しい用途が見つかっていないか

### A/Bテストの実施

新しい指示を試す際は、別のアカウントやセッションで比較してみましょう。

```
【パターンA】現在のカスタム指示
【パターンB】新しいカスタム指示

同じ質問をして、どちらがより有用な回答を返すか比較
```

```
## コーディング規約

- 複数の選択肢がある場合は必ずトレードオフを明示
```

## まとめ

ChatGPTのカスタム指示は、AI活用の生産性を飛躍的に向上させる強力な機能です。

### 重要ポイントの振り返り

1. **基本設定をしっかり行う**
   - 自分の職業・スキルレベルを明確に
   - 希望する回答スタイルを具体的に指定

2. **職種に合わせてカスタマイズ**
   - エンジニアならコード規約を
   - マーケターなら数値・ROIを意識
   - ライターなら読者層とトーンを明確に

3. **定期的に見直す**
   - スキルや興味の変化に合わせて更新
   - 効果測定を忘れずに

4. **チームで活用する場合は標準化**
   - 共通のカスタム指示テンプレートを作成
   - 定期的に振り返りと改善
