## 1. トークン数を意識したプロンプト設計

### なぜ重要か

OpenAI APIの料金体系は、主に**入力トークン数と出力トークン数**で決まります。GPT-4の場合、入力1Kトークンで$0.03、出力1Kトークンで$0.06かかります(2024年時点)。

無駄なトークンを削減するだけで、コストは劇的に下がります。

### 具体的な実装例

```python
# ❌ 悪い例:冗長なプロンプト
bad_prompt = """
あなたは親切で優秀なアシスタントです。
以下のユーザーの質問に対して、丁寧に詳しく回答してください。
質問の背景も考慮し、可能な限り詳細な情報を提供してください。
また、関連する情報があれば合わせて教えてください。

ユーザーの質問:{question}

上記の質問に対して、分かりやすく丁寧に回答してください。
"""

# ✅ 良い例:簡潔で的確なプロンプト
good_prompt = """
以下の質問に簡潔に回答してください。

質問:{question}
"""
```

### 実践的なコスト削減tips

1. **システムプロンプトは最小限に**: 毎回送信されるため、本当に必要な指示だけに絞る
2. **繰り返しを避ける**: 同じ内容を言い換えて複数回書かない
3. **出力長を制限する**: `max_tokens`パラメータで出力の上限を設定

```python
import openai

response = openai.ChatCompletion.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "system", "content": "簡潔に回答してください。"},
        {"role": "user", "content": question}
    ],
    max_tokens=150,  # 出力を制限
    temperature=0.7
)
```

**削減効果**: この手法だけで平均30-40%のコスト削減を実現

## 2. モデルの使い分け戦略

### タスクに応じた最適なモデル選択

すべてのタスクにGPT-4を使う必要はありません。タスクの複雑度に応じてモデルを使い分けることで、大幅なコスト削減が可能です。

| タスク | 推奨モデル | 理由 |
|--------|-----------|------|
| 簡単な分類・判定 | gpt-3.5-turbo | コストが1/10、速度も速い |
| 要約・抽出 | gpt-3.5-turbo | 十分な精度 |
| 複雑な推論・コード生成 | gpt-4 | 高度な推論が必要 |
| 長文理解 | gpt-4-turbo | コンテキスト長が必要 |

### 実装例:タスクルーティング

```python
def get_appropriate_model(task_type: str) -> str:
    """タスクタイプに応じて最適なモデルを返す"""
    model_mapping = {
        "classification": "gpt-3.5-turbo",
        "summarization": "gpt-3.5-turbo",
        "translation": "gpt-3.5-turbo",
        "complex_reasoning": "gpt-4",
        "code_generation": "gpt-4",
    }
    return model_mapping.get(task_type, "gpt-3.5-turbo")

# 使用例
task = "classification"
model = get_appropriate_model(task)

response = openai.ChatCompletion.create(
    model=model,
    messages=[{"role": "user", "content": prompt}]
)
```

**削減効果**: タスクの70%をGPT-3.5-turboで処理することで、約60%のコスト削減

## 3. キャッシング戦略の導入

### 同じ質問への重複リクエストを防ぐ

ユーザーからの質問には、似たようなものや全く同じものが頻繁にあります。これらをキャッシュすることで、API呼び出しを削減できます。

### Redisを使った実装例

```python
import redis
import json
import hashlib
from openai import OpenAI

client = OpenAI()
redis_client = redis.Redis(host='localhost', port=6379, db=0)

def get_cache_key(messages: list) -> str:
    """メッセージからキャッシュキーを生成"""
    content = json.dumps(messages, sort_keys=True)
    return hashlib.md5(content.encode()).hexdigest()

def get_cached_response(messages: list, model: str = "gpt-3.5-turbo"):
    """キャッシュを確認してからAPI呼び出し"""
    cache_key = f"chatgpt:{model}:{get_cache_key(messages)}"
    
    # キャッシュ確認
    cached = redis_client.get(cache_key)
    if cached:
        print("✅ キャッシュヒット!")
        return json.loads(cached)
    
    # API呼び出し
    response = client.chat.completions.create(
        model=model,
        messages=messages
    )
    
    result = response.choices[0].message.content
    
    # キャッシュに保存(24時間)
    redis_client.setex(
        cache_key,
        86400,  # 24時間
        json.dumps(result)
    )
    
    return result

# 使用例
messages = [
    {"role": "user", "content": "Pythonでリストを逆順にする方法は?"}
]

answer = get_cached_response(messages)
```

### キャッシュ戦略のポイント

1. **適切な有効期限**: 情報の鮮度要求に応じて設定(FAQ:長め、ニュース:短め)
2. **キャッシュキーの設計**: メッセージ内容全体をハッシュ化
3. **キャッシュヒット率の監視**: 効果測定を忘れずに

**削減効果**: キャッシュヒット率30%で、約30%のコスト削減

## 4. ストリーミングとバッチ処理の活用

### ストリーミングによるユーザー体験の向上

ストリーミングを使うと、応答が始まるまでの体感速度が向上し、結果的にタイムアウトによる再試行を減らせます。

```python
def stream_response(messages: list):
    """ストリーミングで応答を取得"""
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=messages,
        stream=True  # ストリーミング有効化
    )
    
    full_response = ""
    for chunk in response:
        if chunk.choices[0].delta.content is not None:
            content = chunk.choices[0].delta.content
            print(content, end="")
            full_response += content
    
    return full_response
```

### バッチ処理で効率化

複数の独立したタスクがある場合、バッチ処理することで効率的に処理できます。

```python
import asyncio
from openai import AsyncOpenAI

async_client = AsyncOpenAI()

async def process_batch(questions: list[str]):
    """複数の質問を並行処理"""
    tasks = []
    for question in questions:
        task = async_client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=[{"role": "user", "content": question}]
        )
        tasks.append(task)
    
    # 並行実行
    responses = await asyncio.gather(*tasks)
    return [r.choices[0].message.content for r in responses]

# 使用例
questions = [
    "Pythonとは?",
    "JavaScriptとは?",
    "Rustとは?"
]

answers = asyncio.run(process_batch(questions))
```

**削減効果**: タイムアウトによる再試行が減り、約10-15%のコスト削減

## 5. Function Callingを活用した効率化

### 構造化されたデータ抽出

従来のプロンプトエンジニアリングでは、出力形式の指定が曖昧になりがちでした。Function Callingを使うことで、必要な情報だけを効率的に抽出できます。

### 実装例:商品情報の抽出

```python
def extract_product_info(text: str):
    """商品レビューから構造化データを抽出"""
    functions = [
        {
            "name": "extract_product_data",
            "description": "商品情報を抽出する",
            "parameters": {
                "type": "object",
                "properties": {
                    "product_name": {
                        "type": "string",
                        "description": "商品名"
                    },
                    "rating": {
                        "type": "integer",
                        "description": "評価(1-5)"
                    },
                    "sentiment": {
                        "type": "string",
                        "enum": ["positive", "neutral", "negative"],
                        "description": "感情"
                    }
                },
                "required": ["product_name", "rating", "sentiment"]
            }
        }
    ]
    
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": f"以下のレビューを分析:{text}"}],
        functions=functions,
        function_call={"name": "extract_product_data"}
    )
    
    return json.loads(
        response.choices[0].message.function_call.arguments
    )

# 使用例
review = "このBluetoothイヤホンは音質が素晴らしい!★5つです。"
result = extract_product_info(review)
# {'product_name': 'Bluetoothイヤホン', 'rating': 5, 'sentiment': 'positive'}
```

### メリット

- **出力の一貫性**: JSON形式で確実に取得
- **後処理が不要**: パースエラーのリスクなし
- **トークン削減**: 不要な説明文が含まれない

**削減効果**: 構造化データ抽出タスクで約20-30%のトークン削減

## コスト監視とアラート設定

削減策を実施しても、継続的な監視がなければ効果は限定的です。

### 監視ダッシュボードの構築

```python
import logging
from datetime import datetime

class APIUsageTracker:
    """API使用量トラッキング"""
    
    def __init__(self):
        self.daily_usage = {
            "requests": 0,
            "input_tokens": 0,
            "output_tokens": 0,
            "cost": 0.0
        }
    
    def track_request(self, response, model: str):
        """リクエストを記録"""
        usage = response.usage
        
        # トークン数
        input_tokens = usage.prompt_tokens
        output_tokens = usage.completion_tokens
        
        # コスト計算(GPT-3.5-turboの場合)
        cost = (input_tokens * 0.0015 / 1000) + \
               (output_tokens * 0.002 / 1000)
        
        # 記録
        self.daily_usage["requests"] += 1
        self.daily_usage["input_tokens"] += input_tokens
        self.daily_usage["output_tokens"] += output_tokens
        self.daily_usage["cost"] += cost
        
        # アラート
        if self.daily_usage["cost"] > 100:  # $100超過
            self.send_alert()
    
    def send_alert(self):
        """コストアラート送信"""
        logging.warning(
            f"⚠️ 日次コストが$100を超えました: ${self.daily_usage['cost']:.2f}"
        )

# 使用例
tracker = APIUsageTracker()

response = client.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[{"role": "user", "content": "Hello"}]
)

tracker.track_request(response, "gpt-3.5-turbo")
```
