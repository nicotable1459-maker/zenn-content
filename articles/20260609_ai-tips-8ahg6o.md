## はじめに

ChatGPTを業務で使い始めたものの、「思ったような回答が得られない」「毎回質問を考えるのが大変」と感じていませんか?

この記事では、私が実際に日々の業務で実践している**プロンプト設計のテクニック**を体系的にまとめました。適切なプロンプト設計により、ChatGPTの出力品質は劇的に向上し、業務効率を大幅に改善できます。

**この記事で得られること:**
- 効果的なプロンプトの構造と設計原則
- 実務ですぐ使える具体的なテンプレート集
- 業務別の実践的な活用事例
- よくある失敗パターンとその対処法

## プロンプト設計の基本原則

### 1. 役割（Role）を明確に定義する

ChatGPTに適切な「役割」を与えることで、回答の質と専門性が大きく向上します。

**悪い例:**
```
議事録を作ってください
```

**良い例:**
```
あなたは10年以上の経験を持つプロジェクトマネージャーです。
以下の会議内容から、アクションアイテムと担当者を明確にした
議事録を作成してください。
```

役割設定により、ChatGPTは適切な文脈と専門用語を用いた回答を生成します。

### 2. コンテキスト（Context）を与える

背景情報を提供することで、より的確な回答が得られます。

**必要なコンテキスト情報:**
- プロジェクトの目的や背景
- 対象読者や利用シーン
- 制約条件（文字数、形式、トーンなど）
- 期待するアウトプットの具体例

**実践例:**
```
【プロジェクト背景】
新規Webサービスの開発プロジェクト（B2B SaaS）

【対象読者】
技術に詳しくない経営層

【制約条件】
- A4用紙1枚（1000文字以内）
- 専門用語は使わず平易な表現で
- 数値データで効果を示す

上記を踏まえ、今月の開発進捗レポートを作成してください。
```

### 3. 出力形式（Format）を指定する

期待する出力形式を明示することで、再利用しやすい回答が得られます。

**形式指定の例:**
- Markdown形式の表
- 箇条書きリスト
- JSON形式
- ステップバイステップの手順
- コードブロック

```
以下の情報を、Markdown形式の表で整理してください。

| 項目 | 内容 | 優先度 | 期日 |
|------|------|--------|------|

という形式で出力してください。
```

【出力形式】
件名:
本文:

```

## 業務シーン別実践例

### ケース1: コードレビューコメント生成

プログラミング業務では、コードレビューに時間がかかることがあります。以下のようなプロンプトで効率化できます。

```python
# プロンプト例
"""
あなたは経験豊富なシニアエンジニアです。
以下のPythonコードをレビューし、改善提案を行ってください。

【レビュー観点】
1. パフォーマンス
2. 可読性
3. セキュリティ
4. エラーハンドリング

【出力形式】
各観点について、問題点と具体的な改善コードを提示してください。

【コード】
def process_data(data):
    result = []
    for item in data:
        if item > 0:
            result.append(item * 2)
    return result
"""
```

### ケース2: カスタマーサポート対応文作成

クレーム対応など、繊細なコミュニケーションが必要な場合のテンプレートです。

```
【役割】
共感力の高いカスタマーサポート担当者

【状況】
[問題の内容]: システム障害により3時間サービスが利用できなかった
[顧客の感情状態]: 非常に怒っている
[事実関係]: 原因はサーバー障害、現在は復旧済み

【対応方針】
1. まず謝罪と共感を示す
2. 事実を正確に説明する
3. 再発防止策を提示する
4. 補償内容を提案する

【制約】
- 言い訳に聞こえる表現は避ける
- 専門用語は平易に言い換える
- 誠意が伝わる温かみのある表現

上記を踏まえた対応文を作成してください。
```

### ケース3: データ分析レポート作成

（3行で要約）

（各項目を見出し付きで）

```

## プロンプトの効果を高める応用テクニック

### テクニック1: チェーン・オブ・ソート（思考の連鎖）

複雑な問題を解決する際は、段階的に思考させることで精度が向上します。

```
以下の問題について、ステップバイステップで考えてください。

【問題】
新規事業の収益性を評価する

【思考プロセス】
1. まず、必要な情報をリストアップしてください
2. 次に、各情報をどう分析すべきか説明してください
3. その上で、判断基準を提示してください
4. 最後に、総合的な評価を行ってください

各ステップで「なぜそう考えたか」も説明してください。
```

### テクニック2: Few-Shot Learning（例示による学習）

期待する出力形式を例示することで、一貫性のある回答が得られます。

```
以下の例に従って、商品説明文を作成してください。

【例1】
商品名: ワイヤレスイヤホン
説明文: 「通勤時間が音楽タイムに変わる。ノイズキャンセリング機能で、
満員電車でもクリアな音質を楽しめます。バッテリー持続時間24時間で、
充電の心配も不要です。」

【例2】
商品名: スマートウォッチ
説明文: 「健康管理が習慣になる。心拍数、睡眠、運動量を自動記録。
スマホ通知も手元で確認でき、大切な連絡を見逃しません。」

【作成依頼】
商品名: [新商品名]
特徴: [特徴を列挙]

上記の例と同じトーンとスタイルで説明文を作成してください。
```

### テクニック3: 制約条件の明示

明確な制約を設けることで、実用的な回答が得られます。

```
【必須条件】
- 実装期間: 2週間以内
- 予算: 10万円以内
- 使用技術: Python, PostgreSQL
- メンバー: 2名（フルスタックエンジニア）

【除外条件】
- 外部APIの新規契約が必要なもの
- インフラの大幅な変更が必要なもの

上記制約の中で実現可能な機能改善案を5つ提案してください。
```

## よくある失敗パターンと対処法

### 失敗パターン1: 曖昧すぎる指示

**NG例:**
```
良い記事を書いてください
```

**問題点:** 「良い」の定義が不明確

**改善案:**
```
【対象読者】初心者エンジニア
【目的】AWS入門の理解促進
【文字数】2000字
【含めるべき要素】
- 実際のコード例
- 図解またはイメージ
- よくあるエラーと対処法

上記を満たすブログ記事を作成してください。
```

### 失敗パターン2: 一度に多くを求めすぎる

**NG例:**
```
マーケティング戦略、実行計画、予算配分、KPI設定、
リスク分析を全部やってください
```

**問題点:** 出力が浅く広くなりすぎる

**改善案:**
まず全体構造を作り、次に各要素を深掘りする段階的アプローチ

```
# ステップ1
マーケティング戦略の骨子を3つの柱で提案してください

# ステップ2（最初の出力を受けて）
提案された戦略1について、詳細な実行計画を作成してください
```

### 失敗パターン3: コンテキストの不足

**NG例:**
```
この問題を解決してください
[問題の説明のみ]
```

**問題点:** 背景情報がなく、的外れな回答になる

**改善案:**
```
【背景】
SaaS製品の解約率が3ヶ月で15%→25%に上昇

【これまでの対策】
- カスタマーサポートの強化（効果なし）
- オンボーディング改善（小幅改善）

【制約】
- 予算: 月50万円
- 期間: 3ヶ月で効果を出す

【求める提案】
上記を踏まえ、解約率を15%に戻すための施策を提案してください
```

## プロンプト管理のベストプラクティス

### 1. テンプレート化して再利用する

よく使うプロンプトはテンプレート化し、Notionやスプレッドシートで管理しましょう。

**管理項目例:**
- プロンプト名
- 用途・目的
- テンプレート本文
- 変数部分（[XXX]で明示）
- 使用例
- 作成日・更新日

### 2. バージョン管理する

プロンプトを改善した際は、前のバージョンも残しておきます。

```
# v1.0 - 2024/01/15
[初期バージョン]

# v1.1 - 2024/02/01
[
