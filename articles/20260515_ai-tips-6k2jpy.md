## なぜClaude 3.5 Sonnetが要約に最適なのか

### 長文理解能力の圧倒的優位性

Claude 3.5 Sonnetは最大200,000トークン（約15万語）のコンテキストウィンドウを持ち、複数の長文ドキュメントを同時に処理できます。これは以下のような利点をもたらします：

- **PDFや長大なMarkdownファイルを丸ごと投入可能**
- 文脈を完全に保持した要約が可能
- 複数文書間の関連性を理解した統合要約

## 基本プロンプト設計の黄金パターン

効果的な要約プロンプトは、以下の5要素で構成されます：

### 1. 役割（Role）の明確化

```
あなたは経験豊富なビジネスアナリストです。
複雑な情報を整理し、意思決定に必要なポイントを抽出することに長けています。
```

役割設定により、AIの「視点」が定まり、出力の一貫性が向上します。

### 2. タスク（Task）の具体化

```
以下の文書を要約してください。要約は以下の構造で出力してください：
1. 3行サマリー（最重要ポイント）
2. 主要な論点（箇条書き5項目）
3. データ・数値の要約
4. 推奨アクション
```

### 3. 制約（Constraints）の設定

```
制約条件：
- 全体で500文字以内
- 専門用語は必ず説明を付ける
- 主観的な解釈は避け、事実のみを記載
```

### 4. コンテキスト（Context）の提供

```
【背景情報】
この文書は四半期の営業報告書です。
対象読者は経営陣で、意思決定のための情報が必要です。
```

### 5. 出力形式（Format）の指定

```
出力形式：
- Markdown形式
- 見出しレベルは##から開始
- 重要度に応じて🔴🟡🟢の絵文字を使用
```

## 要約精度を劇的に高める5つのテクニック

### テクニック1：段階的要約（チャンク処理）

超長文ドキュメント（10万語以上）の場合、以下のアプローチが効果的：

```python
# Python実装例
import anthropic

def chunked_summarization(document, chunk_size=50000):
    """段階的要約の実装"""
    client = anthropic.Anthropic(api_key="your-api-key")
    
    # ステップ1: 文書を分割
    chunks = split_document(document, chunk_size)
    
    # ステップ2: 各チャンクを要約
    summaries = []
    for i, chunk in enumerate(chunks):
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=2000,
            messages=[{
                "role": "user",
                "content": f"""
以下は文書の第{i+1}部分です。主要ポイントを箇条書きで要約してください。

【文書】
{chunk}
"""
            }]
        )
        summaries.append(response.content[0].text)
    
    # ステップ3: 要約を統合
    final_summary = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=3000,
        messages=[{
            "role": "user",
            "content": f"""
以下は文書の各部分の要約です。これらを統合して、
全体の包括的な要約を作成してください。

【部分要約】
{chr(10).join(summaries)}
"""
        }]
    )
    
    return final_summary.content[0].text
```
