# プロンプトエンジニアリングとは？

プロンプトエンジニアリングとは、AIに対する指示（プロンプト）を最適化することで、より精度の高い回答を得るための技術です。

プログラミングにおけるコーディング技術と同じように、AIとのコミュニケーションにも「型」や「ベストプラクティス」が存在します。

## なぜプロンプトが重要なのか

AIは与えられた情報の範囲内で回答を生成します。つまり：

- **曖昧な指示 → 曖昧な回答**
- **具体的な指示 → 具体的で実用的な回答**

この法則を理解するだけで、AIの活用効果は飛躍的に向上します。

# 回答精度を高める5つの基本原則

## 1. 役割（ロール）を明確に指定する

AIに「あなたは〇〇の専門家です」と役割を与えると、その視点で回答してくれます。

### ❌ 悪い例
```
Pythonのコードを教えて
```

### ✅ 良い例
```
あなたは10年以上の経験を持つPythonエンジニアです。
初心者にも分かりやすく、ベストプラクティスに沿った
コードを教えてください。
```

**効果：** 専門性の高い、実践的な回答が得られる

---

## 2. 出力形式を具体的に指定する

「どのような形式で回答してほしいか」を明示すると、そのまま使える成果物が得られます。

### 実例：ブログ記事の構成案を作る場合

```
以下の条件でブログ記事の構成案を作成してください：

【テーマ】リモートワークの生産性向上
【対象読者】リモートワーク初心者
【文字数】2000文字程度
【出力形式】
- 見出し（H2, H3）
- 各セクションの概要（50文字程度）
- 想定文字数

表形式で出力してください。
```

**効果：** すぐに活用できる、具体的なアウトプットが得られる

---

## 3. 制約条件を明示する

「やってほしいこと」だけでなく、「やってほしくないこと」も伝えましょう。

### 実例：コードレビュー依頼

```
以下のPythonコードをレビューしてください。

【制約条件】
- セキュリティの観点を重視
- パフォーマンスの問題を指摘
- 一般的な説明は不要、具体的な改善案のみ記載
- 指摘は3つまでに絞る

[コードを貼り付け]
```

**効果：** 焦点が絞られた、実用的なフィードバックが得られる

---

## 4. 段階的に指示を出す（Chain of Thought）

複雑なタスクは、ステップに分けて指示すると精度が上がります。

### 実例：競合分析レポート作成

```
以下の手順で競合分析を行ってください：

ステップ1: 競合企業A社の強みを3つ列挙
ステップ2: 各強みに対する我が社の現状を評価
ステップ3: 対策案を優先順位付きで提案

各ステップごとに回答を待ちます。
まずステップ1から始めてください。
```

**効果：** 思考プロセスが可視化され、より深い分析が得られる

---

## 5. 例示（Few-shot Learning）を活用する

「こういう感じで」と例を示すと、AIはパターンを学習して同じ形式で回答します。

### 実例：商品説明文の生成

```
以下の例を参考に、新商品の説明文を作成してください：

【例1】
商品名：ワイヤレスイヤホンPro
説明文：通勤時間を上質な音楽体験に。ノイズキャンセリング機能搭載で、
あなただけの静寂空間を作り出します。最大24時間の連続再生で、
一日中音楽と共に。

【例2】
商品名：スマートウォッチLite
説明文：健康管理を、もっと身近に。心拍数・睡眠・運動量を自動記録。
スマホと連携して、あなたのライフスタイルをスマートにサポートします。

---

【新商品】
商品名：ポータブル加湿器Mini
対象：デスクワーク中心のビジネスパーソン
特徴：USB給電、超音波式、静音設計
```

**効果：** ブランドトーンを統一した、一貫性のある成果物が得られる

# よくある失敗例と改善策

## 失敗例1：質問が抽象的すぎる

❌ **悪い例**
```
良いコードの書き方を教えて
```

✅ **改善例**
```
Reactの関数コンポーネントで、パフォーマンスを考慮した
コードを書くためのベストプラクティスを、
具体例（useMemo、useCallbackの使い分け）と共に教えてください。
```

---

## 失敗例2：一度に複数のことを聞く

❌ **悪い例**
```
PythonとJavaScriptの違いを教えて。
あと、どっちが初心者向けで、将来性があるのか、
学習方法も知りたい。
```

✅ **改善例**
```
まず最初に：
PythonとJavaScriptの主な違いを、
「構文」「用途」「実行環境」の3点で比較してください。

その後、別の質問として初心者向け度合いについて聞きます。
```

---

## 失敗例3：文脈を省略しすぎる

【状況】Next.js 13のApp Routerを使用
【問題】ページ遷移時に状態が保持されない
【やりたいこと】カート情報を遷移後も保持したい

以下のコードを修正してください：
[コード]
```

| 用途 | おすすめツール | 理由 |

# プロンプトの品質を高める継続的な改善

## PDCAサイクルを回す

1. **Plan（計画）**: プロンプトを設計
2. **Do（実行）**: AIに投げて結果を得る
3. **Check（評価）**: 期待通りか確認
4. **Act（改善）**: プロンプトを修正

## 自分用のプロンプトライブラリを作る

効果的だったプロンプトは、NotionやScrapboxなどに保存しておきましょう。

```markdown
# プロンプトライブラリ

## カテゴリ：コードレビュー
### テンプレート名：セキュリティ重視レビュー
[プロンプト本文]

### 使用場面
- API実装時
- 外部入力を扱うコード

### 効果
SQLインジェクション等の脆弱性を指摘してくれる
```
