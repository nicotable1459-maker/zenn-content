## はじめに：タスク管理に疲れていませんか?

毎日のタスク管理、うまくいっていますか？

「ToDoリストが溢れて何から手をつければいいか分からない」
「優先順位をつけたはずなのに、結局緊急対応に追われて終わる」
「タスク管理ツールを使っているのに、なぜか効率が上がらない」

こんな悩みを抱えている方は多いのではないでしょうか。私も以前は同じでした。

この記事では、私が試行錯誤の末にたどり着いた**「3つの箱」メソッド**をご紹介します。このシンプルな方法を実践することで、1日30分のタスク整理時間で生産性が劇的に向上しました。

**この記事で得られること：**
- タスク管理の新しい視点と具体的な手法
- すぐに実践できるシンプルなフレームワーク

## 「3つの箱」メソッドとは？

「3つの箱」メソッドは、すべてのタスクを以下の3つのカテゴリに分類して管理する手法です：

### 📥 Box 1：Inbox（受信箱）
- **役割**：すべてのタスク・アイデア・依頼を一時的に受け止める
- **ルール**：何も考えず、とにかく全部ここに入れる
- **頻度**：随時（思いついたら即座に）

### 🎯 Box 2：Today（今日やる箱）
- **役割**：今日確実に完了させるタスクのみを格納
- **ルール**：最大5〜7個まで。それ以上は入れない
- **頻度**：1日1回（朝の15分間）

### 🗄️ Box 3：Someday（いつかやる箱）
- **役割**：重要だが緊急ではないタスク・プロジェクトを保管
- **ルール**：週次レビューで見直し、必要に応じてTodayへ移動
- **頻度**：週1回の見直し（金曜午後など）

この3つの箱を使い分けることで、**「考える時間」と「実行する時間」を完全に分離**できます。

## なぜこの方法が効果的なのか？

### 1. 認知負荷の軽減

人間の脳は同時に複数のことを考えると疲弊します。「3つの箱」メソッドでは：

- Inboxに入れる時は**分類を考えない**（ただ記録するだけ）
- Todayを見る時は**実行だけに集中**できる
- Somedayがあることで**「忘れてしまう不安」から解放される**

この分離により、常に「今何をすべきか」が明確になります。

### 2. 決断疲れの防止

1日に何度も「これは今やるべきか？」と判断していると、決断疲れ（Decision Fatigue）が蓄積します。

このメソッドでは：
- タスクの分類は**1日1回、朝の15分だけ**
- それ以外の時間は**Todayリストを淡々と消化**
- 緊急タスクが来ても、一旦Inboxへ入れて判断を先延ばし可能

### 3. 完了感の獲得

Todayの箱を小さく保つことで：
- 毎日「今日やることは全部終わった」という達成感を得られる
- 終わらないToDoリストによるストレスが激減
- モチベーションの維持が容易になる

## 実践方法：朝の15分ルーティン

具体的な運用方法を紹介します。

### Step 1：Inboxの整理（5分）

前日からInboxに溜まったタスクを確認し、以下のいずれかに振り分けます：

```
判断基準：
✅ 今日やる必要がある → Todayへ
📅 いつかやりたい → Somedayへ
🗑️ もう不要 → 削除
⏱️ 2分以内で終わる → 即実行
```

**ポイント：** 2分ルール（2分以内で終わるタスクはその場で片付ける）を活用すると、小さなタスクが溜まりません。

### Step 2：Todayリストの作成（5分）

Inboxから移動したタスクと、Somedayから今日やるべきタスクを選び、Todayリストを作ります。

**重要な制約：**
- **最大7個まで**（理想は5個）
- 合計所要時間が実労働時間の70%以内
- 残り30%は緊急対応・突発作業用のバッファ

例：
```markdown
## Today (2024/01/15)
1. [ ] A社への提案書作成（2h）
2. [ ] 月次レポート提出（1h）
3. [ ] チームミーティング準備（30m）
4. [ ] Bさんへのフィードバック（30m）
5. [ ] 来週のプレゼン資料アウトライン作成（1h）

合計: 5時間 → 8時間勤務の62.5%
```

### Step 3：優先順位の明確化（5分）

Todayリスト内で優先順位をつけます。私は以下の方法を使っています：

```
🔴 最重要：絶対に今日完了させる（1-2個）
🟡 重要：できる限り今日完了させる（2-3個）
🟢 余力があれば：明日に回しても問題ない（1-2個）
```

重要度は以下の基準で判断：
- 締め切りが今日か？
- 他者の作業をブロックしているか？
- 先延ばしにすると問題が大きくなるか？

## デジタルツールでの実装例

### Notionでの実装

Notionデータベースを使った実装例です：

```
データベース名: Task Manager

プロパティ：
- Name（タスク名）
- Status（Select）: Inbox / Today / Someday / Done
- Priority（Select）: 🔴High / 🟡Medium / 🟢Low
- Estimate（Number）: 所要時間（分）
- Date（Date）: 実行予定日

ビュー：
1. Inbox View（Status = Inbox）
2. Today View（Status = Today、Dateでソート）
3. Someday View（Status = Someday、Priorityでグループ化）
4. Archive（Status = Done、過去7日間）
```

**自動化のヒント：**
- Statusが"Done"に変わったら自動的にArchiveへ移動
- 毎朝9時にTodayビューの通知を送る
- 週末にSomedayビューのリマインダーを設定

### Trelloでの実装

よりビジュアルな管理が好きな方にはTrelloもおすすめです：

```
ボード名: Daily Task Flow

リスト構成：
1. 📥 Inbox
2. 🎯 Today
3. ⏳ In Progress（作業中の視覚化用）
4. ✅ Done Today
5. 🗄️ Someday

カード移動ルール：
- 朝：Inbox → Today（最大7枚）
- 作業開始：Today → In Progress
- 完了：In Progress → Done Today
- 夜：Done Todayをアーカイブ
```

### アナログ（紙とペン）での実装

デジタルが苦手な方、あるいは手書きの方が集中できる方向けの方法：

**用意するもの：**
- A4ノート1冊（1ページを3分割して使用）
- 付箋（3色：白・黄・青）

**運用方法：**
```
左1/3: Inbox（白付箋を貼る）
中1/3: Today（黄付箋を貼る）
右1/3: Someday（青付箋を貼る）

毎朝：
1. Inboxの付箋を確認
2. 今日やるものをTodayエリアに移動
3. それ以外をSomedayまたは廃棄
4. 完了したらTodayの付箋を剥がす
```

手書きの利点は「物理的に移動させる」ことで達成感が得られること、デジタル通知に邪魔されないことです。

## よくある失敗パターンと対策

### 失敗1：Todayに詰め込みすぎる

**症状：**
「全部重要だから」と10個も15個もTodayに入れてしまい、結局半分も終わらない。

**対策：**
- **強制的に7個制限**を設ける（物理的に7つまでしか書けないフォーマットを使う）
- 「今日やらないと本当に困ること」だけを選ぶ
- 残りは全てSomedayへ（「やらない」ではなく「今日はやらない」）

### 失敗2：Inboxを空にしない

**症状：**
Inboxが100個以上溜まり、整理するのが億劫になって放置。

**対策：**
- **Inbox Zero を目指す**：毎朝必ずInboxを空にする
- 判断に迷ったら「とりあえずSomeday」でOK
- 週次レビューで古いものは定期的に削除

### 失敗3：Somedayが墓場になる

**症状：**
Somedayに入れたタスクを二度と見ない。結果的に重要なタスクが埋もれる。

**対策：**
- **毎週金曜日15時を「週次レビュー」の時間に固定**
- Somedayを見直し、3つまでTodayに昇格させる
- 3ヶ月触っていないタスクは削除（本当に必要なら再度浮上する）

### 失敗4：緊急タスクで崩壊する

**症状：**
急な依頼が来るたびにTodayリストを無視して対応し、計画が台無しに。

**対策：**
- 緊急タスクも**一旦Inboxへ**記録
- 5分だけ時間を取り、本当に今日必要か判断
- Todayに追加する場合、代わりに別のタスクをSomedayへ移動（総数を維持）
- 30%のバッファ時間を必ず確保しておく

## 応用テクニック：さらに効率を上げる

### テクニック1：テーマデーの設定

曜日ごとにテーマを決めると、Todayリストの作成が楽になります：

```
月曜：会議・コミュニケーション中心
火曜：深い集中作業（執筆・設計など）
水曜：チーム協業・レビュー
木曜：深い集中作業
金曜：週次レビュー・小タスク処理
```

### テクニック2：時間ブロッキングとの併用

Todayリストのタスクをカレンダーに実際にブロックすることで、実行可能性が高まります：

```
09:00-09:15 朝の整理タイム
09:15-11:15 A社提案書作成 🔴
11:15-12:15 月次レポート 🟡
13:00-13:30 チームMTG準備 🟡
13:30-14:30 （バッファ時間）
14:30-15:00 Bさんフィードバック 🟡
15:00-16:00 プレゼン資料アウトライン 🟢
16:00-17:00 （バッファ時間 / Inbox処理）
```

### テクニック3：エネルギーレベルとのマッチング

タスクの難易度と自分のエネルギーレベルを合わせます：

```
午前（エネルギー高）：
→ 創造的タスク、重要な判断を要するタスク

午後前半（エネルギー中）：
→ ルーチン作業、コミュニケーション

午後後半（エネルギー低）：
→ 簡単な事務作業、メール返信、Inbox整理
```

### テクニック4：達成ログの記録

Done（完了）したタスクを記録することで、自己効力感を高められます：

```markdown
## Done Today (2024/01/15)
✅ A社提案書完成（想定2h → 実績1.5h）
✅ 月次レポート提出（想定1h → 実績1h）
✅ チームミーティング実施
✅ Bさんへ建設的フィードバック提供
✅ プレゼンアウトライン完成

感想：A社提案書が予定より早く終わり、達成感
