## はじめに：なぜプロンプトエンジニアリングが重要なのか

ChatGPTを使っているけれど、期待した回答が得られない――そんな経験はありませんか?

実は、ChatGPTの性能を最大限に引き出すには「プロンプトエンジニアリング」というスキルが不可欠です。同じ質問でも、聞き方を変えるだけで回答の質が劇的に向上します。

本記事では、実務で即座に使える7つのプロンプトテクニックを、具体例とともに紹介します。これらを実践すれば、あなたの業務効率は確実に向上するでしょう。

**この記事で得られること:**
- プロンプトの基本構造と効果的な書き方
- 業務で使える7つの実践テクニック
- すぐにコピペして使えるプロンプトテンプレート
- よくある失敗例とその改善方法

## プロンプトエンジニアリングの基本原則

### 1. 明確な役割(Role)を与える

ChatGPTに具体的な役割を与えることで、回答の方向性が定まります。

**悪い例:**
```
Pythonのコードをレビューしてください。
```

**良い例:**
```
あなたは10年以上の経験を持つシニアPythonエンジニアです。
以下のコードをセキュリティ、パフォーマンス、可読性の観点からレビューしてください。
```

役割を明確にすることで、ChatGPTは適切な専門性を持った回答を生成します。

### 2. 文脈(Context)を提供する

背景情報を与えることで、より的確な回答が得られます。

**悪い例:**
```
メールの文章を書いてください。
```

**良い例:**
```
私はスタートアップのCTOです。
エンジニア採用の面接で不採用となった候補者に対して、
前向きで丁寧なフィードバックメールを書きたいです。
候補者は技術力は高かったものの、チームフィットの観点で見送りとなりました。
```

## 実践テクニック7選

### テクニック1: ゼロショット→フューショットへの進化

**ゼロショット（例なし）:**
```
ブログ記事のタイトルを考えてください。
```

**フューショット（例あり）:**
```
以下のような形式でブログ記事のタイトルを5つ考えてください。

例:
- 「エンジニアが知っておくべき○○の秘訣」
- 「初心者でもできる！○○完全ガイド」
- 「○○で生産性が3倍になった話」

テーマ: プロンプトエンジニアリング
```

具体例を示すことで、出力のフォーマットや品質を制御できます。

### テクニック2: Chain of Thought（思考の連鎖）

複雑な問題を解決する際は、段階的に考えさせることが効果的です。

```
以下の問題をステップバイステップで解いてください。

問題: あるECサイトで、カート放棄率が60%です。
この問題を改善するための施策を考えてください。

手順:
1. まず、カート放棄の一般的な原因を列挙してください
2. 次に、それぞれの原因に対する解決策を考えてください
3. 最後に、優先順位をつけて実装ロードマップを提案してください
```

このテクニックにより、ChatGPTはより論理的で構造化された回答を生成します。

### テクニック3: 制約条件を明示する

出力の形式や条件を明確に指定することで、期待通りの結果が得られます。

```
以下の制約に従ってPythonコードを書いてください。

【制約条件】
- Python 3.9以上で動作すること
- 外部ライブラリは標準ライブラリのみ使用
- 関数には型ヒントを必ず付ける
- docstringを含める
- エラーハンドリングを実装する
- コメントは日本語で記述

【タスク】
CSVファイルを読み込み、特定の列でフィルタリングして
結果を新しいCSVファイルとして保存する関数を作成してください。
```

### テクニック4: ペルソナベースプロンプト

複数の視点から意見を得たい場合に有効です。

```
以下の新機能について、3つの異なるペルソナの視点から意見をください。

機能: アプリに「AIによる記事要約機能」を追加する

ペルソナ1: 30代ビジネスパーソン（時間効率重視）
ペルソナ2: 60代のシニアユーザー（操作のシンプルさ重視）
ペルソナ3: 20代の学生（無料で使えることを重視）

各ペルソナが感じる利点と懸念点をそれぞれ3つずつ挙げてください。
```

### テクニック5: 反復改善プロンプト

初回の出力を改善するための追加指示を行います。

**1回目:**
```
Reactでのパフォーマンス最適化について記事を書いてください。
```

**2回目（改善指示）:**
```
この記事に以下を追加してください:
- 具体的なコード例を各セクションに追加
- 実測データ（パフォーマンス改善の数値）を含める
- よくある落とし穴とその回避方法を追加
- Next.jsでの応用例も含める
```

### テクニック6: テンプレート化プロンプト

頻繁に使うタスクはテンプレート化しておきましょう。

**コードレビューテンプレート:**
```
以下のコードをレビューしてください。

【コード】
{ここにコードを貼り付け}

【レビュー観点】
1. セキュリティ上の問題
2. パフォーマンスの改善点
3. コードの可読性
4. ベストプラクティスとの照合
5. テストの必要性

【出力形式】
- 各観点ごとに項目立てて記載
- 重要度を「高・中・低」で評価
- 具体的な改善コード例を提示
```

### テクニック7: メタプロンプト（プロンプトを生成させる）

ChatGPT自身に最適なプロンプトを考えさせることもできます。

```
私は以下のタスクを実行したいと考えています:
「技術ブログ記事を効率的に執筆するためのアシスタントとして機能してほしい」

このタスクを達成するために最も効果的なプロンプトを作成してください。
プロンプトには、役割、制約条件、出力形式などを含めてください。
```

## よくある失敗パターンと改善方法

### 失敗例1: 曖昧すぎる指示

❌ **悪い例:**
```
良い記事を書いてください。
```

✅ **改善例:**
```
技術ブログ記事を書いてください。

【要件】
- 対象読者: Reactの基礎を学んだ初級〜中級者
- 文字数: 2,000〜3,000文字
- テーマ: カスタムフックの作成方法
- 含めるべき要素: コード例、図解の提案、よくあるエラーとその対処法
- トーン: 親しみやすく、かつ専門的
```

### 失敗例2: コンテキスト不足

❌ **悪い例:**
```
この問題を解決してください。
エラーが出ます。
```

✅ **改善例:**
```
以下のエラーを解決したいです。

【環境】
- OS: macOS Ventura 13.4
- Python: 3.11.2
- ライブラリ: pandas 2.0.0

【エラー内容】
```
AttributeError: 'DataFrame' object has no attribute 'append'
```

【試したこと】
- pandasを最新版にアップデート
- 公式ドキュメントを確認

【期待する回答】
- エラーの原因説明
- 具体的な修正コード
- pandas 2.0での推奨される代替方法
```

### 失敗例3: 一度に多くを求めすぎる

複雑なタスクは分割して依頼しましょう。

❌ **悪い例:**
```
ECサイトを作ってください。フロントエンド、バックエンド、
データベース設計、デプロイ方法、テストコード、
セキュリティ対策、パフォーマンス最適化まで全部お願いします。
```

✅ **改善例:**
```
ECサイトの開発を段階的に進めたいです。
まずは第1フェーズとして、データベース設計から始めたいと思います。

【第1フェーズ: データベース設計】
- 必要なテーブル: Users, Products, Orders, OrderItems
- 各テーブルのカラム定義
- リレーション設計
- インデックス設計

ER図をMermaid記法で作成し、CREATE TABLE文も併せて提供してください。

※次のステップでバックエンドAPIの設計に進む予定です。
```
