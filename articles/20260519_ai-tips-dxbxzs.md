## はじめに：なぜプロンプトが重要なのか

ChatGPTを使っているけれど、期待した答えが返ってこない。同じ質問でも人によって結果が全く違う――そんな経験はありませんか?

実は、**ChatGPTの性能の80%はプロンプト(指示文)の質で決まる**と言われています。この記事では、実務で即活用できるプロンプトエンジニアリングのテクニックを7つ紹介します。

### この記事で得られること

- AIから正確で有用な回答を引き出す具体的手法
- 業務効率を劇的に向上させるプロンプトパターン
- すぐに使えるテンプレート集
- よくある失敗例とその改善策

## 1. 役割(Role)を明確に指定する

AIに「あなたは〇〇の専門家です」と役割を与えることで、回答の質と専門性が大きく向上します。

### ❌ 悪い例
```
TypeScriptのコードをレビューしてください
```

### ✅ 良い例
```
あなたは10年以上の経験を持つシニアTypeScript開発者です。
以下のコードについて、型安全性、パフォーマンス、保守性の観点から
詳細にレビューしてください。

[コード]
```

### 効果的な役割設定のポイント

- **具体的な専門性**を示す（「エンジニア」ではなく「フロントエンド開発に特化したシニアエンジニア」）
- **経験年数**や**実績**を付加する
- **視点や価値観**を指定する（コスト重視、ユーザー体験重視など）

## 2. 文脈(Context)を十分に提供する

AIは文脈を理解することで、より的確な回答を生成します。背景情報を惜しみなく提供しましょう。

### 実例：マーケティング施策の提案

```
【背景】
- B2B SaaS企業（従業員50名）
- 主要顧客：中小企業の人事部門
- 現在の課題：リード獲得コストが高騰（CPA 50,000円）
- 予算：月額30万円
- 期間：3ヶ月

【目標】
CPAを30,000円以下に削減しつつ、月間50件のリード獲得

上記の状況で、最も効果的なマーケティング施策を3つ提案してください。
各施策について、実施方法、予想効果、リスクを含めてください。
```

### 文脈に含めるべき要素

1. **現状**：今どういう状況か
2. **制約**：予算、時間、リソースなどの制限
3. **目標**：何を達成したいのか
4. **優先順位**：何を重視するか

## 3. 出力形式(Format)を指定する

回答の形式を明示することで、そのまま使える実用的なアウトプットが得られます。

### フォーマット指定の例

```
以下の情報を表形式でまとめてください：

| 項目 | 説明 | 重要度(1-5) | 実装難易度 |
|------|------|-------------|------------|
| ... | ... | ... | ... |
```

### よく使うフォーマット指定

**箇条書き形式**
```
以下の形式で3つ提案してください：
- [提案名]: [一行説明]
  - メリット: 
  - デメリット:
  - 実装期間:
```

**JSON形式**（プログラムで処理する場合）
```json
{
  "recommendations": [
    {
      "title": "提案タイトル",
      "priority": "high",
      "estimatedTime": "2 weeks",
      "requiredResources": ["人材", "ツール"]
    }
  ]
}
```

## 4. 段階的思考(Chain of Thought)を促す

複雑な問題では、AIに「段階的に考えさせる」ことで精度が向上します。

### テクニック：「ステップバイステップで考えてください」

```
以下のビジネス課題について、ステップバイステップで分析してください：

【課題】
ECサイトのカート放棄率が75%と高い

【分析手順】
1. 考えられる原因を列挙
2. 各原因の影響度を評価
3. 優先的に対処すべき原因を特定
4. 具体的な改善策を提案
5. 実施スケジュールを作成

各ステップを丁寧に考察してください。
```

### 数値計算の例

```
【問題】
あるサービスの月間利用者が10,000人で、
コンバージョン率が2%、顧客単価が3,000円です。
広告費を月30万円かけた場合のROIを計算してください。

**計算過程も含めて**段階的に示してください。
```

## 5. Few-Shot Learning（例示による学習）

具体例を示すことで、AIは期待する回答スタイルを理解します。

### 実践例：メール文面の生成

```
以下の例に倣って、顧客への謝罪メールを作成してください。

【例1 - 配送遅延の場合】
件名：お荷物の配送遅延に関するお詫び

〇〇様

平素より格別のご愛顧を賜り、誠にありがとうございます。

この度は、ご注文いただいた商品の配送が遅延し、
ご迷惑をおかけしておりますことを深くお詫び申し上げます。
[具体的な理由と対応]

【例2 - システム障害の場合】
[類似の例]

---

【今回のケース】
- 状況：請求書の金額誤り
- 顧客：法人取引先
- 影響：10,000円の過剰請求
```

## 6. 制約と禁止事項を明示する

「やってはいけないこと」を指定することで、不要な回答を排除できます。

### 制約の指定例

```
新規事業のアイデアを5つ提案してください。

【制約条件】
- 初期投資：500万円以内
- 収益化まで：6ヶ月以内
- 必要人員：3名以内

【除外事項】
- 飲食業は除く
- 物理的な店舗が必要なビジネスは除く
- 法規制が厳しい業界（金融、医療など）は除く

【求める情報】
各アイデアについて、市場規模、競合状況、実現可能性を含めてください。
```

## 7. 反復と改善（Iterative Refinement）

一度の質問で完璧な答えを求めず、対話を通じて精度を高めていきます。

### 効果的な反復パターン

**第1段階：全体像の把握**
```
ブログ記事「AIツールの選び方」の構成案を提案してください
```

**第2段階：詳細化**
```
「2. 目的別AIツールの分類」のセクションを、
各カテゴリの具体例を3つずつ含めて詳しく展開してください
```

**第3段階：最適化**
```
上記の内容を、初心者にも分かりやすいように、
専門用語を減らして書き直してください
```

## 実践テンプレート集

すぐに使えるテンプレートをいくつか紹介します。

### テンプレート1：技術調査レポート

```
あなたは[技術分野]に詳しいテックリードです。

【調査対象】
[技術名/ツール名]

【調査項目】
1. 概要と主な特徴
2. 他の選択肢との比較（最低3つ）
3. 導入メリット・デメリット
4. 学習コスト（習得に必要な時間）
5. コミュニティの活発さ
6. 本番環境での採用実績

【出力形式】
技術選定会議で使用するため、各項目を簡潔にまとめてください。
表形式も活用してください。
```

### テンプレート2：コードレビュー

```
あなたは[言語名]のベテラン開発者です。
以下のコードを以下の観点でレビューしてください：

【コード】
[コードを貼り付け]

【レビュー観点】
1. ✅ 良い点（具体的に）
2. ⚠️ 改善すべき点（優先度付き）
3. 🐛 潜在的なバグ
4. 💡 リファクタリング提案
5. 📚 参考となるベストプラクティス

各項目について、理由を含めて説明してください。
改善案がある場合は、修正後のコードも提示してください。
```

## よくある失敗例と改善策

### 失敗例1：質問が漠然としている

❌ **悪い例**
```

```

```
【状況】
- 対象：経営層（5名）
- 時間：15分
- 目的：新規AIツール導入の承認を得る
- 予算規模：年間300万円

効果的なプレゼン構成と、各スライドに入れるべき情報を教えてください。
経営層が意思決定に必要とする情報を重点的に含めてください。
```

## プロンプト最適化のチェックリスト

実際にプロンプトを書く際は、以下をチェックしてください：

- [ ] **役割**：AIに適切な専門性を与えたか？
- [ ] **文脈**：背景情報は十分か？
- [ ] **目標**：達成したいことが明確か？
- [ ] **制約**：条件や制限を示したか？
- [ ] **形式**：期待する出力形式を指定したか？
- [ ] **具体例**：必要に応じて例を示したか？
- [ ] **除外**：やってほしくないことを明示したか？
- [ ] **検証**：回答の検証方法を考えたか？

## まとめ：プロンプトエンジニアリングは技術である

この記事で紹介した7つのテクニックを実践することで、ChatGPTから得られる価値は飛躍的に向上します。

### 重要ポイントの再確認

1. **役割設定**で専門性を引き出す
2. **十分な文脈**で的確な回答を得る
3. **出力形式**でそのまま使える成果物を得る
4. **段階的思考**で複雑な問題を解決する
5. **例示**で期待するスタイルを伝える
6. **制約明示**で不要な回答
