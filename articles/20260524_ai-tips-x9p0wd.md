# はじめに

ChatGPTを使っているけれど、「いまいち期待した答えが返ってこない」「もっと活用できるはずなのに...」と感じていませんか?

実は、ChatGPTの回答品質は**プロンプト(質問文)の書き方次第で劇的に変わります**。同じ質問でも、少し工夫するだけで、より具体的で実用的な回答を引き出せるのです。

この記事では、日々の業務でChatGPTを活用している私が実践している、**今日から使える具体的なプロンプトテクニック**を厳選してご紹介します。

## この記事で得られること

- ChatGPTから高品質な回答を引き出すための5つの基本テクニック
- 実務でそのまま使える具体的なプロンプト例
- 用途別(コーディング、文章作成、アイデア出し等)の実践テクニック
- よくある失敗パターンとその改善方法

それでは、実践的なテクニックを見ていきましょう!

## 1. 役割(ロール)を明確に指定する

ChatGPTに**具体的な役割を与える**ことで、回答の方向性や専門性を大きく向上させることができます。

### ❌ 悪い例
```
Reactについて教えてください。
```

### ✅ 良い例
```
あなたは10年以上の実務経験を持つシニアフロントエンドエンジニアです。
React初心者に対して、Reactの基本概念を実例を交えながら分かりやすく説明してください。
```

### なぜ効果的なのか

役割を指定することで:
- **回答の専門性**が向上する
- **対象読者**に合わせた説明になる
- **具体例**を含めた実践的な内容になる

### 実務で使えるロール例

```
# コードレビュー時
あなたは厳格なコードレビュアーです。
以下のコードの問題点、改善点、セキュリティリスクを指摘してください。

# ドキュメント作成時
あなたはテクニカルライターです。
非エンジニアにも理解できる技術ドキュメントを作成してください。

# 企画立案時
あなたは経験豊富なプロダクトマネージャーです。
市場分析を踏まえた新機能の提案書を作成してください。
```

## 2. 出力形式を具体的に指定する

ChatGPTに**どのような形式で出力してほしいか**を明確に伝えることで、そのまま使える回答を得られます。

### テーブル形式で整理

```
以下の情報を表形式で整理してください:
- 列: 技術名、メリット、デメリット、学習コスト
- 対象: React, Vue.js, Angular
```

実際の出力例:

| 技術名 | メリット | デメリット | 学習コスト |
|--------|----------|------------|------------|
| React | 豊富なエコシステム、求人が多い | 設定が複雑 | 中 |
| Vue.js | 学習曲線が緩やか、日本語情報が豊富 | エンタープライズ事例が少ない | 低 |
| Angular | フルスタックフレームワーク | 学習コストが高い | 高 |

### ステップバイステップ形式

```
初心者がDockerを学ぶためのロードマップを、
以下の形式で作成してください:

## ステップ1: [タイトル]
- 学ぶ内容:
- 推奨リソース:
- 目安期間:

(以下同様に5ステップまで)
```

### コードブロック形式

```
TypeScriptでユーザー認証機能を実装するコードを、
以下の要件で作成してください:
- JWT認証
- Express.js使用
- エラーハンドリング含む
- コメント付き
- 実行可能なコード
```

## 3. 制約条件を明確にする

「何をしてほしいか」だけでなく、**「何をしてほしくないか」**も伝えることが重要です。

### 実践例: 記事執筆

```
以下の条件でブログ記事を書いてください:

【必須条件】
- 文字数: 1500-2000字
- 対象読者: プログラミング初心者
- トーン: フレンドリーで分かりやすい
- 構成: 見出し3つ以上

【禁止事項】
- 専門用語の多用(使う場合は必ず解説)
- 抽象的な説明のみ
- コード例なしの説明
```

### 実践例: コード生成

```
Pythonで簡易的なToDoアプリのコードを書いてください。

【要件】
- CLI版でOK
- データはJSON形式でファイル保存
- 追加・削除・一覧表示機能
- 100行以内

【制約】
- 外部ライブラリ不使用(標準ライブラリのみ)
- Python 3.8以降で動作
- エラーハンドリング必須
```

## 4. Few-shot Learning: 例を示す

ChatGPTに**具体例を与える**ことで、求めているアウトプットのイメージを正確に伝えられます。

### ネーミング生成の例

```
以下の例を参考に、新しいプロジェクト名を5つ提案してください:

【既存の例】
- QuickStart → 素早く始められることを表現
- DataFlow → データの流れを管理するツール
- CodeSculpt → コードを彫刻するように整形

【新規プロジェクト概要】
APIドキュメント自動生成ツール
```

### 文章のトーン調整

```
以下の文章と同じトーンで、別の技術記事の導入部を書いてください:

【例文】
「TypeScript、使ってますか?『型定義がめんどくさい』『学習コストが高そう』
そんな声をよく聞きますが、一度慣れてしまえば、もうJavaScriptには戻れません。
今回は、TypeScript初心者が最初につまずきがちなポイントを、
実例とともに解説していきます!」

【今回のテーマ】
Docker入門
```

## 5. 段階的に質問を深める(連鎖プロンプト)

一度に完璧な答えを求めるのではなく、**段階的に質問を深めていく**ことで、より洗練された回答を得られます。

### 実践フロー

```
【第1段階: ブレインストーミング】
「eコマースサイトのパフォーマンス改善方法を10個リストアップしてください」

↓ ChatGPTが10個の案を提示

【第2段階: 絞り込み】
「この中で、実装コストが低く効果が高いものを3つ選び、
優先順位をつけてください」

↓ ChatGPTが3つに絞り込んで優先順位付け

【第3段階: 具体化】
「1位の施策について、実装手順を具体的に教えてください。
コード例も含めてお願いします」

↓ 詳細な実装ガイドを取得
```

この方法のメリット:
- **段階的に精度が上がる**
- **途中で方向修正できる**
- **最終的に実用的な答えが得られる**

## よくある失敗パターンと改善方法

### 失敗例1: 曖昧すぎる質問

❌ **悪い例**
```
いいコードの書き方を教えて
```

✅ **改善例**
```
Pythonでクリーンなコードを書くための具体的なテクニックを、
以下の観点から教えてください:
- 命名規則
- 関数の設計
- コメントの書き方
各項目について、悪い例と良い例のコード比較を含めてください。
```

### 失敗例2: 前提情報の不足

❌ **悪い例**
```
このエラーどうすればいい?
TypeError: undefined is not a function
```

✅ **改善例**
```
React(v18.2) + TypeScriptプロジェクトで以下のエラーが発生しました:

【エラー】
TypeError: undefined is not a function at handleClick (App.tsx:25)

【該当コード】
[コードを貼り付け]

【発生タイミング】
ボタンクリック時

【環境】
- Node.js: v18.16.0
- React: 18.2.0
- TypeScript: 5.0.4
```

### 失敗例3: 一度に詰め込みすぎ

❌ **悪い例**
```
ReactとVue.jsとAngularの違いを説明して、
それぞれのメリット・デメリットと、
初心者におすすめの学習順序と、
各フレームワークのチュートリアルも教えて
```

✅ **改善例**
```
【第1回】
React、Vue.js、Angularの主な違いを、
比較表形式で簡潔に教えてください。

→ 回答を得た後

【第2回】
この3つの中で、JavaScript初心者が最初に学ぶべきものと、
その理由を教えてください。

→ さらに深掘り
```

## まとめ: 高品質な回答を得るためのチェックリスト

ChatGPTを効果的に活用するために、以下のポイントを押さえましょう:

### ✅ プロンプト作成前のチェックリスト

- [ ] **役割(ロール)を指定**したか?
- [ ] **出力形式を具体的に**指定したか?
- [ ] **制約条件や禁止事項**を明確にしたか?
- [ ] **必要な前提情報**をすべて含めたか?
- [ ] **例示**で求める内容を具体化したか?
- [ ] 質問は**一度に1つのテーマ**に絞られているか?

### 🚀 さらなるステップアップのために

1. **プロンプトを記録する**: 効果的だったプロンプトをストックしておく
2. **A/Bテストする**: 同じ質問を複数の方法で試してみる
3. **コミュニティから学ぶ**: 他の人のプロンプト例を参考にする
4. **定期的に見直す**: ChatGPTのアップデートで最適なプロンプトも変わる
