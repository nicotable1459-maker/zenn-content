## プロンプトエンジニアリングとは?

プロンプトエンジニアリングとは、AIに対して**最適な指示(プロンプト)を設計する技術**です。同じ質問でも、聞き方次第で回答の質は大きく変わります。

### 悪い例と良い例の比較

**❌ 悪い例:**
```
レポートを書いて
```

**✅ 良い例:**
```
あなたは経験豊富なビジネスアナリストです。
以下の条件で市場分析レポートを作成してください。

【対象市場】: 日本のSaaS市場
【期間】: 2020年〜2024年
【文字数】: 1500文字程度
【含めるべき内容】:
- 市場規模の推移
- 主要プレイヤー
- 今後の成長予測
【形式】: 経営層向けのエグゼクティブサマリー形式
```

このように、**役割・条件・形式**を明確にすることで、求める回答に近づきます。

## 効果的なプロンプトの5つの基本原則

### 1. 役割(ロール)を明確に設定する

ChatGPTに「あなたは〜です」と役割を与えることで、その視点からの回答が得られます。

```
あなたは10年以上の経験を持つシニアエンジニアです。
初心者向けにGitの使い方を説明してください。
```

**効果的な役割設定の例:**
- あなたは経験豊富なキャリアコンサルタントです
- あなたはベストセラー作家です
- あなたはデータサイエンティストです
- あなたは小学生にも分かりやすく教える教師です

### 2. 具体的な制約条件を与える

曖昧な指示ではなく、**数値や形式で明確に**条件を指定しましょう。

```
以下の条件でブログ記事の導入文を書いてください:
- 文字数: 200-300文字
- トーン: フレンドリーで親しみやすい
- ターゲット: プログラミング初心者
- 含めるキーワード: Python, 学習方法, 挫折しない
```

### 3. 段階的に指示を与える(Chain of Thought)

複雑なタスクは、**ステップバイステップで思考させる**ことで精度が上がります。

```
以下の手順で分析してください:

Step 1: 提供されたデータから主要な傾向を3つ抽出してください
Step 2: それぞれの傾向について、考えられる原因を列挙してください
Step 3: 最も重要な傾向を1つ選び、その理由を説明してください
Step 4: 具体的な改善提案を3つ提示してください
```

### 4. 出力形式を明示する

回答の**フォーマットを指定**することで、そのまま使える形で出力されます。

```
以下の形式のJSON形式で出力してください:

{
  "title": "記事タイトル",
  "summary": "要約(100文字以内)",
  "tags": ["タグ1", "タグ2", "タグ3"],
  "estimated_reading_time": "読了時間(分)"
}
```

### 5. 例を示す(Few-shot Learning)

**期待する出力の例**を1〜2個示すことで、パターンを学習させます。

```
以下の例に倣って、商品レビューを要約してください。

【例1】
元レビュー: 「配送は早かったですが、商品の品質がイマイチでした。値段相応かな。」
要約: 配送◎ 品質△ コスパ普通

【例2】
元レビュー: 「期待以上の品質で大満足!デザインも良く、毎日使っています。」
要約: 品質◎ デザイン◎ 満足度高

【要約してください】
元レビュー: 「説明書が分かりにくく、初期設定に苦労しました。でも一度設定できれば快適です。」
```

## 実践的なプロンプトテンプレート集

### テンプレート1: コードレビュー依頼

```
あなたは経験豊富なシニアエンジニアです。
以下のコードをレビューし、改善点を指摘してください。

【レビュー観点】
- コードの可読性
- パフォーマンス
- セキュリティ
- ベストプラクティスへの準拠

【コード】
```python
# ここにコードを貼り付け
```

【出力形式】
1. 良い点
2. 改善すべき点(重要度順)
3. 改善後のコード例
```

### テンプレート2: 企画書・提案書作成

```
あなたはビジネス企画のプロフェッショナルです。
以下の情報をもとに、新規事業企画書を作成してください。

【事業概要】: [ここに概要を記載]
【ターゲット】: [ターゲット層]
【解決する課題】: [課題内容]

【企画書に含める項目】
1. エグゼクティブサマリー
2. 市場分析
3. 競合分析
4. 収益モデル
5. 実行計画(3ヶ月、6ヶ月、1年)
6. 必要リソース

【文字数】: 各項目300文字程度
【トーン】: 経営層向けの説得力ある表現
```

### テンプレート3: 文章の添削・リライト

```
以下の文章を、指定した条件でリライトしてください。

【元の文章】
[ここに文章を貼り付け]

【リライト条件】
- ターゲット読者: [例: ビジネスパーソン、学生など]
- トーン: [例: フォーマル、カジュアル、専門的]
- 文字数: [例: 元の80%に圧縮、2倍に拡張]
- 改善ポイント: [例: より分かりやすく、説得力を高める]

【出力】
1. リライト後の文章
2. 変更した主なポイント(箇条書き)
```

## 高度なテクニック

### テクニック1: ペルソナ設定の活用

複数の視点からの意見が欲しい場合、異なるペルソナを設定します。

```
以下のビジネスアイデアについて、3つの異なる立場から評価してください:

【アイデア】: [ビジネスアイデアを記載]

【評価者1】: 保守的な投資家の視点
【評価者2】: 革新的なスタートアップ起業家の視点
【評価者3】: エンドユーザーの視点

各視点から、メリット・デメリット・懸念点を述べてください。
```

### テクニック2: 自己批判プロンプト

ChatGPTに**自分の回答を批判させる**ことで、より洗練された結果を得られます。

```
# 第1段階
[通常のプロンプト]

# 第2段階
上記の回答について、以下の観点から批判的に検討してください:
- 論理的な矛盾はないか
- 見落としている重要な視点はないか
- より良いアプローチはないか

そして、改善版を提示してください。
```

### テクニック3: コンテキストの累積活用

長い会話の中で**段階的に情報を追加**していくことで、精度を高めます。

```
# 最初のプロンプト
新しいWebサービスのアイデアを考えています。
ターゲットは「忙しいビジネスパーソン」です。

# 2つ目(回答を受けて)
なるほど。では「タスク管理」に焦点を当てたいと思います。
既存のTrello、Asanaとの差別化ポイントを考えてください。

# 3つ目
「AI自動優先度付け」という機能に興味があります。
この機能の具体的な実装アイデアを教えてください。
```

## 実際の業務での活用例

### 活用例1: 議事録の自動生成

```
以下の会議の文字起こしテキストから、議事録を作成してください。

【会議名】: 週次定例会議
【日時】: 2024年1月15日 10:00-11:00
【参加者】: 田中、佐藤、鈴木

【文字起こしテキスト】
[ここに文字起こしテキストを貼り付け]

【議事録形式】
1. 決定事項
2. TODO(担当者、期限付き)
3. 次回までの宿題
4. 保留事項

簡潔に箇条書きでまとめてください。
```

### 活用例2: メールの下書き

```
以下の状況に適したビジネスメールを作成してください。

【状況】: クライアントへの納期遅延の謝罪
【理由】: 想定外の技術的課題が発生
【新しい納期】: 当初より2週間遅れ
【トーン】: 誠実で信頼を損なわない表現

【含めるべき要素】
- 状況説明
- 謝罪
- 遅延の理由(簡潔に)
- 新しいスケジュール
- 再発防止策
- 今後の対応

【文字数】: 300-400文字
```

### 活用例3: データ分析レポート

```
以下のCSVデータを分析し、インサイトをレポートにしてください。

【データ】
[CSVデータを貼り付け、またはデータの概要を説明]

【分析の観点】
- 全体的な傾向
- 異常値や特徴的なパターン
- 時系列での変化
- セグメント間の比較

【レポート形式】
1. サマリー(3行)
2. 主要な発見事項(Top 3)
3. 詳細分析
4. 推奨アクション

可能であれば、グラフや表の形式も提案してください。
```

## よくある失敗パターンと対策

### 失敗パターン1: 曖昧すぎる指示

❌ **悪い例**: 「良い感じにして」「適当に書いて」
✅ **改善策**: 具体的な基準を数値や例で示す

### 失敗パターン2: 一度に多くを求めすぎる

❌ **悪い例**: 10個のタスクを1つのプロンプトに詰め込む
✅ **改善策**: タスクを分割し、段階的にアプローチする

### 失敗パターン3: コンテキスト不足

❌ **悪い例**: 背景情報なしで専門的な判断を求める
✅ **改善策**: 必要な前提情報をすべて提供する

### 失敗パターン4: 出力形式の未指定

❌ **悪い例**: 形式を指定せず、使いにくい回答を受け取る
✅ **改善策**: 表、JSON、Markdownなど明確に指定する
