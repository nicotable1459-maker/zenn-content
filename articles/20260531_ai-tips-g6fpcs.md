## はじめに

AIアシスタントを使っているけれど、「なんとなく質問して、なんとなく答えをもらう」だけになっていませんか?

本記事では、**Claude 3.5 Sonnetを実務で最大限活用するための、即実践できるプロンプトテンプレート**を5つ厳選して紹介します。これらのテンプレートを使うことで、コーディング、ドキュメント作成、コードレビュー、リファクタリングなどの作業時間を大幅に短縮できます。

## テンプレート1: コード生成の黄金パターン

### 基本テンプレート

```
# 役割
あなたは[言語/フレームワーク]の経験豊富なエンジニアです。

# タスク
[具体的に実装したい機能]を実装してください。

# 制約条件
- [言語/フレームワーク]のバージョン: [バージョン]
- 使用するライブラリ: [ライブラリ名]
- コーディング規約: [規約があれば]
- エラーハンドリングを含める
- テストコードも生成する

# 出力形式
1. 実装コード
2. 使用例
3. 注意点
```

### 実践例: React コンポーネント生成

```
# 役割
あなたはReactとTypeScriptの経験豊富なフロントエンドエンジニアです。

# タスク
ユーザー検索フォームのコンポーネントを実装してください。
検索ボックスに入力するとリアルタイムでフィルタリングされる仕様です。

# 制約条件
- React 18 + TypeScript
- 使用するライブラリ: なし(標準機能のみ)
- Hooksを使用する
- デバウンス処理を実装(300ms)
- アクセシビリティを考慮(aria属性)
- エラーハンドリングを含める

# 出力形式
1. コンポーネントコード
2. 型定義
3. 使用例
4. パフォーマンス最適化のポイント
```

### このテンプレートのポイント

- **役割設定**: AIに専門性を持たせることで、より適切な回答を引き出す
- **具体的な制約**: バージョンやライブラリを明示することで、環境に合ったコードを生成
- **出力形式の指定**: 欲しい情報を確実に得られる

### 実践例

```
# 役割
あなたは経験豊富なPythonコードレビュアーです。

# タスク

# レビュー観点
1. セキュリティ脆弱性
2. エラーハンドリング
3. パフォーマンス
4. Pythonic な書き方

# コード
```python
def login(username, password):
    user = db.query("SELECT * FROM users WHERE username='" + username + "'")
    if user and user.password == password:
        token = generate_token(user.id)
        return {"token": token}
    return {"error": "Invalid credentials"}
```

# 出力形式
各問題点に対して、具体的なコード例を含めた改善案を提示してください。
```

```

# 含めるべき内容
1. 概要説明
2. 使用方法

# 出力形式
Markdown形式で、読みやすく構造化されたドキュメント
```

### 実践例: API ドキュメント生成

```
# 役割
あなたはRESTful APIドキュメントのライティング専門家です。

# タスク
以下のFastAPIエンドポイントに対するAPIドキュメントを作成してください。

# 対象読者
このAPIを利用する外部の開発者

# コード
```python
@app.post("/api/v1/users/{user_id}/posts")
async def create_post(
    user_id: int,
    post: PostCreate,
    current_user: User = Depends(get_current_user)
):
    if current_user.id != user_id:
        raise HTTPException(status_code=403)
    return await post_service.create(user_id, post)
```

# 含めるべき内容
1. エンドポイント概要
2. 認証要件
3. リクエスト仕様(パス、ボディ、ヘッダー)
4. レスポンス仕様(成功時、エラー時)
5. curlでの実行例
6. レスポンスサンプル(JSON)

# 出力形式
OpenAPI形式を意識したMarkdownドキュメント
```

### このテンプレートの応用

- README.md の自動生成
- JSDoc / Docstring の生成
- ユーザーマニュアルの作成
- チュートリアルの作成

```
# 役割

# 問題状況

# 環境情報
- OS: Ubuntu 22.04
- Node.js: v20.10.0
- フレームワーク: Express 4.18.2

# エラーメッセージ
```
FATAL ERROR: Reached heap limit Allocation failed - JavaScript heap out of memory
```

# 問題のコード
```javascript
const cache = {};

app.get('/api/data/:id', (req, res) => {
  const id = req.params.id;
  if (!cache[id]) {
    cache[id] = fetchDataFromDB(id);
  }
  res.json(cache[id]);
});
```

# 期待する動作
キャッシュを使って高速にレスポンスを返す

# 実際の動作
時間経過とともにメモリ使用量が増加し続ける

# 出力形式
1. メモリリークの原因
2. 修正コード(LRUキャッシュ等の実装)
3. メモリ管理のベストプラクティス
4. 監視・検知方法
```

## 効果を最大化する3つのコツ

### 1. コンテキストを十分に与える

AIは与えられた情報からしか判断できません。以下を明確にしましょう:

- プロジェクトの目的・規模
- 技術スタック
- 制約条件(パフォーマンス、セキュリティなど)
- チームのコーディング規約

### 2. 段階的に詳細化する

いきなり完璧を求めず、対話しながら改善していきます:

```
1. まず大まかな設計を確認
2. 詳細実装を依頼
3. エッジケースや最適化を追加
4. テストケースを生成
```

### 3. 出力形式を具体的に指定する

「わかりやすく説明して」ではなく:

```
# 出力形式
- 箇条書きで3つのポイントにまとめる
- 各ポイントにコード例を1つずつ付ける
- 最後に表形式で比較表を作成する
```
